# 2. About Paper

Most of the existing data augmentation methods are heuristic, so there are two problems:

1. Optimal augmentation depends on domain and finding it is time-consuming
2. Heuristic data augmentation often creates 'garbage' data

TANDA learns to find the optimal augmentation for that dataset.

Learning is very similar to Neural Architecture Search (NAS).

![TANDA Pipeline](../../../../img/210320_TANDA/Figure_1.png)
![Search Space](../../../../img/210320_TANDA/Figure_2.png)

1. Heuristically choose augmentation functions (rotation, parallel movement, ...), magnitude (for example, 10 degrees, 30 degrees, ...), and probability (how many augmentation functions will be used)
2. Generator creates Transformation Functions (TF) on the search space mentioned in 1
3. Augment the data using the generated TF
4. Discriminator classify whether input is augmented or not

In other words, the generator learns to create TF that makes augmented data distribution similar to original data.
(augmented data with TF is similar in distribution to actual data)

![Loss](../../../../img/210320_TANDA/Figure_3.png)

To prevent the generator from learning inverse combinations (left flip + right flip), diatance is added in the loss function.
(distance considering both raw input space and feature space of augmented, unaugmented data)

2 models can be used as a generator model.

1. Independent model : mean field model
2. State-based model : LSTM

Since the process of creating TF is not differentiable, it is learned using reinforcement learning.

Discriminator uses the CNN used by DCGAN.
